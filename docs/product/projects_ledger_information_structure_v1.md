# 项目台账信息架构 v1

## 1. 页面目标

`projects.ledger` 从“项目卡片集合”升级为“项目群管理台账”。

## 2. 信息层次

### 2.1 第一层：项目群总览（新增）

- 在建项目数
- 预警项目数
- 已完工项目数
- 项目群规模（项目数）

该层位于卡片区之前，先帮助管理者判断“整体是否可控”。

### 2.2 第二层：项目卡片（保留）

卡片优先展示：

- 项目名称
- 状态（中文化）
- 负责人
- 关键金额（`dashboard_invoice_amount`）

并通过状态语义色为高风险/异常项目提供轻量提示。

## 3. 状态语义映射（示例）

- `draft` -> 草稿
- `01_in_progress` / `in_progress` / `open` -> 进行中
- `done` -> 已完成
- `blocked` / `overdue` / `risk` -> 风险
- `cancel` / `cancelled` -> 已取消

## 4. 交互兼容性

- 项目卡片点击跳转行为保持不变；
- 不修改底层 action/scene 路由协议；
- 仅在展示层增加总览条和状态可读性。
