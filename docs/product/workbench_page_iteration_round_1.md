# 工作台页面迭代方案（Round 1）

## 目标

- 将首跳工作台从“能力与调试信息并列展示”收敛为“行动入口 + 系统提醒 + 状态总览 + 常用功能”的产品化工作台。
- 保持 Scene / Capability / Suggested Action 机制不变，仅优化表达层与信息结构。

## 页面定位

工作台不是静态展示页，而是用户登录后的第一决策面板。

核心回答三件事：

1. 今天要处理什么（今日行动）
2. 系统正在提醒什么（系统提醒）
3. 当前总体状态如何（项目总体状态）

## 信息架构（Round 1）

按用户心理顺序组织：

1. 今日行动（最高优先级）
2. 系统提醒（风险/异常）
3. 项目总体状态（指标与进展）
4. 常用功能（场景与能力入口）

## 本轮改动范围

- 调整工作台区块标题与排序语义。
- 将“系统建议”统一为“系统提醒”。
- 将“经营分析”统一为“项目总体状态”。
- 将“能力入口”统一为“常用功能”。
- 在用户主视图移除 `filters/result_summary/active_filters` 调试型折叠块。

## 本轮不做

- 不修改 scene registry / governance / delivery policy。
- 不修改 ACL / 权限基线 / 认证链路。
- 不重构 page contract envelope。
- 不改 Suggested Action 协议，仅调整入口呈现层。

## 验收标准

- 打开工作台首屏，首先看到“今日行动 + 系统提醒”。
- 页面中不再出现用户不可理解的技术摘要字段。
- “项目总体状态”语义可读，不暴露技术字段命名。
- “常用功能”作为末段入口，不干扰首屏判断。

## Round 1 · Batch 2（已执行）

- 角色焦点块顺序统一前置为：`today_actions`、`risk_alert_panel`、`metric_row_core`。
- 工作台 section 顺序调整为“今日行动 -> 系统提醒 -> 核心指标”。
- `ops` 相关文案业务化：
  - `项目经营概览` -> `项目总体状态`
  - `成本执行率` -> `成本控制率`
  - `资金支付比例` -> `资金支付率`
- 核心指标标签业务化（合同履约率、产值完成率、成本控制率、资金支付率），移除“可见导航节点”类技术导向指标。

## Round 1 · Batch 3（收口）

- 压缩首屏价值表达：`hero.lead` 与 `product_tags` 精简，避免首屏信息过载。
- 去除 `ds_hero` 技术键直出，统一转为“标签 + 值”展示项（角色、默认入口、更新时间、运行状态）。
- 统一区块与卡片高度节奏：
  - zone 间距和块间距统一增大（提升呼吸感）
  - todo/alert/entry/metric/progress 卡片增加最小高度，避免参差。

## Round 1 · Batch 4（模型对齐）

- 进一步对齐 Action -> Alert -> Status -> Capability：
  - 提升 `today_focus` zone 优先级，降低 `hero` zone 优先级；
  - `系统提醒`前置到 `today_focus`（风险提醒 + 建议补充）。
- 用户主视图移除技术感字段展示：
  - 不再展示 `updated_at/status_notice/status_detail` 风格内容；
  - `ds_hero` 改为简化的人类可读摘要行。
