# Login 页面设计说明（Round 1）

## 页面目标

- 将登录页从“通用后台登录框”升级为“智能施工企业管理平台正式入口”。
- 在不改动认证内核的前提下，提高品牌识别度、企业产品感与可用性。

## 目标用户第一印象

- 这是施工企业管理平台，不是默认后台模板。
- 页面稳重、克制、可依赖，符合工程行业管理软件气质。
- 登录动作反馈明确，错误提示清晰，不需要猜测系统状态。

## 文案口径

- 标题：`登录系统`
- 品牌名：`智能施工企业管理平台`
- 副标题：`面向施工企业的项目、合同、成本、资金与风险协同平台`
- 口号：`让项目透明、风险可见、决策有据`
- 字段：`账号` / `密码`
- 按钮：`登录` / `系统正在登录，请稍候…`
- 错误：
  - `账号或密码错误，请重新输入`
  - `网络异常，请稍后重试`
  - `登录失败，请稍后重试`

## 视觉风格关键词

- 稳重
- 企业级
- 工程管理场景
- 低饱和背景渐变
- 轻量层次（光影/胶囊标签）

## 本轮实现点

- 增加品牌信息区（产品名 + 副标题 + 口号）。
- 新增能力提示条（项目、合同/成本、资金、风险）。
- 本地化登录文案与输入占位文案。
- 优化输入框与按钮的 `hover/focus/disabled/loading` 状态。
- 新增用户友好的错误归一化映射，避免直接暴露技术错误。
- 优化背景层次和卡片结构，并补齐移动端收敛样式。

## Round 2 精修（收口）

- 放大登录卡片宽度与内边距，提升入口页气场与信息呼吸感。
- 将 `SCEMS v1.0` 徽标收敛到卡片品牌区，建立稳定从属关系。
- 增加低对比度工程网格背景层，减少左侧“空白感”。
- 将能力标签压缩为短语义并网格化呈现，提升扫读效率。
- 调整“平台名/动作标题”层级：品牌主名更稳，登录动作标题作为次级引导。

## Round 3 入口页重构（品牌区 + 登录区）

- 页面结构升级为双区：左侧品牌叙事区，右侧登录动作区。
- 左侧承载平台定位、价值主张与核心能力卡，不再将能力信息堆叠在登录卡片内。
- 右侧登录卡片保持简洁动作路径（版本信息、登录标题、账号密码、登录按钮）。
- 引入低对比工程网格背景（`opacity` 级别）增强行业语义，避免花哨视觉。
- 新增底部产品版权信息，强化“正式产品入口”感知。

## 本轮不做

- 不修改登录 API、token/session、路由守卫、app.init、后端认证逻辑。
- 不引入复杂动画、重插画或 2C 风格特效。
- 不引入新的认证方式（如短信、SSO、多因子）。

## 后续可迭代项

- 登录页支持可配置品牌（集团/子公司主题）。
- 根据租户展示不同帮助入口（运维电话/使用手册）。
- 加入更细粒度错误码映射（受后端返回结构约束）。
