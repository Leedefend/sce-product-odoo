# 工作台目标与模型（v1）

## 1. 页面使命

工作台不是导航页，也不是调试看板。工作台的唯一使命是：

- 让用户在 10 秒内知道“今天先做什么”；
- 让用户在 30 秒内知道“哪里有风险、是否影响本周目标”；
- 让用户可以一跳进入处理，不在页面里二次找入口。

一句话定义：**工作台 = 行动入口 + 风险提醒 + 状态总览**。

## 2. 信息层级（首屏优先级）

### P0（必须首屏）

- 今日行动：待审批、待确认、待跟进，必须可点击进入处理场景。
- 系统提醒：仅保留高优先风险，不展示泛化建议。

### P1（首屏可见，但权重次于 P0）

- 项目总体状态：4 个经营指标 + 进展摘要，用于判断态势。

### P2（降级展示）

- 常用功能：作为“下一步”入口，不抢占 P0 位置。
- 核心关注（角色摘要）：降级为补充信息区，不作为首要信息。
- 调试字段：仅在 HUD/开发模式可见。

## 3. 动作优先规则

行动项排序采用三因子：

- 风险等级（高风险优先）
- 时效（即将到期/逾期优先）
- 影响范围（影响金额/项目范围优先）

当数据不足时，允许使用“场景行动模板”兜底，但必须保持可执行（可跳转）。

## 4. 本轮落地约束

- 不改 ACL、权限、记录规则。
- 不改 scene registry / governance / delivery policy 核心机制。
- 不重构 contract envelope，仅调整编排与语义。
- 保持 verify 主链路可通过。

## 5. 验收标准（产品侧）

- 用户打开工作台后，首屏第一块必须是“今日行动”。
- 风险提醒必须在首屏，且文案可直接理解。
- 用户无需滚动即可看到“先做什么 + 为什么做 + 去哪里做”。
- 页面不再出现 result_summary/active_filters 等开发字段。
