# 前端产品化设计基线 v1

## 产品定位

普通业务用户面对的是智能施工项目管理产品；配置、发布、使用分析、场景健康和支持诊断属于管理员域。业务页面不展示契约、模型、字段池、trace 或 action id。当前以桌面企业系统为主，移动端先支持查看、待办、审批和轻量录入。

## 信息架构

普通用户一级导航：角色首页、我的工作、项目管理、合同成本、资金管理、现场执行、报表/经营分析。管理员入口单独位于配置中心、发布管理、使用分析、场景健康、支持诊断，不与业务一级导航同权。

## Token 基线

统一入口为 `frontend/apps/web/src/styles/design-system.css` 与 `product-patterns.css`，页面不得新增随机颜色和尺寸。

| Token | 基线 |

| 间距 | 8px 基础体系：8/16/24/32 |

## 页面模板

角色首页使用标题、上下文、待办摘要、主内容、状态反馈；我的工作使用标题、待办筛选、列表和返回路径；业务列表使用 PageHeader、ContextIndicator、FilterBar、DataTable、状态反馈；详情/办理表单使用标题、上下文、主操作、表单主体、校验和返回；配置页面使用管理员上下文、配置内容、发布/诊断反馈。

每页最多一个主操作。保存草稿、返回、取消为次级；删除、撤销为危险；低频动作进入“更多”；提交与保存同时出现时提交为主操作；忙碌时主操作显示进行状态并禁止重复提交。

## 付款申请列表首个切片

页面标题为“付款申请”，标题下显示当前公司和项目范围；主操作为“新建付款申请”；筛选区提供搜索、状态和重置；结果区显示数量、金额右对齐、统一状态标签和行级查看；公司切换显示刷新反馈。空数据、筛选无结果、接口失败和无权限分别使用中文标题、说明、主操作和返回路径，技术详情默认隐藏。

## 安全与兼容

前端只消费权威 `release_navigation_v1`/`delivery_engine_v1`，明确空导航不回退到 `result.nav`。不修改 system.init、权限规则、业务 API、金额或状态语义；暗色主题继续复用现有 token。
