# Form/List Scene Template

## 适用场景

- 列表/表单主线场景：立项、合同、申请、台账等业务流程页面。

## 标准结构

- `list`：筛选、排序、分页、批量动作。
- `form`：分区布局、关键字段组、提交动作、状态流转。
- `policy`：角色可见性与动作权限。

## 最低产品化要求（R2 -> R3）

- R2：list/form 页面结构完整，支持主流程动作。
- R3：补充状态机、校验规则、异常处理与审批联动。

## 检查清单

- list/filter/sort/page 是否契约化。
- form 是否存在关键字段校验与状态约束。
- 是否定义主要动作的可见性与权限策略。
